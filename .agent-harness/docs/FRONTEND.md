# FRONTEND.md

For full product design direction, use `.agent-harness/docs/UI_UX.md`.

## Frontend Rule

Frontend work should prove product clarity, owner control, and generated-surface
safety before adding visual flourish.

Future AI/business-agent products should aim for premium futuristic SaaS, not
cyberpunk. Real product UX comes first; futuristic style, motion, and 3D are
supporting details.

## Required Checks

For meaningful UI changes:

- Verify desktop and mobile layouts.
- Confirm text does not overlap or truncate in primary flows.
- Confirm risk is not communicated by color alone.
- Confirm synthetic, mock, or dry-run states are visible.
- Confirm owner approval is obvious before any risky action.
- Confirm every recommendation can expose evidence or a reason.

## Generated UI Boundary

Future generated surfaces must flow through validated specs and allowlisted
components.

Do not let generated content define arbitrary executable UI code without a
separate sandbox plan, threat model, and review.
