# Refinement Stages

Move ideas through these stages before creating Linear work.

## 1. Raw Idea

Capture the initial thought without over-filtering.

Output:

- One-sentence concept.
- Target user guess.
- Problem guess.
- Promising angles.
- Early risks.

## 2. Problem Clarity

Make the problem specific.

Output:

- Target user.
- Pain point.
- Current workaround.
- Why the problem matters.
- Desired business outcome.

## 3. Workflow Clarity

Describe what the user actually does.

Output:

- Trigger.
- Main workflow.
- Decisions.
- Approvals.
- Success state.
- Failure or blocked states.

## 4. Product Shape

Turn the workflow into a product boundary.

Output:

- MVP capabilities.
- Non-goals.
- Required data.
- UX direction.
- Operational risks.

## 5. Work Breakdown

Break the product shape into buildable chunks.

Output:

- Epics.
- Dependencies.
- Implementation sequence.
- Draft issues.

## 6. Ready For Linear

Only this stage should create promotable issues.

Output:

- Small Linear issues.
- Acceptance criteria.
- Verification.
- Out-of-scope notes.
- Deferred ideas preserved outside the issue list.
