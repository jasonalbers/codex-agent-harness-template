# Product Sense

This repository's product is the agent harness itself.

Good changes make it easier to:

- Add a future project with little manual setup.
- Understand what an agent is allowed to do.
- Trace work from Linear issue to GitHub PR.
- Verify work before merge.
- Recover from failed agent runs.
- Give future products a clear UI/UX starting point without putting product code
  in this base.

Avoid adding product-specific workflows here. Put those in the target project
repo after the base harness is ready.
