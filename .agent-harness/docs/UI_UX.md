# UI/UX Guidance

This base repository does not include a product UI. It does include reusable
guidance for future products that start from this template.

Use this document when a Linear issue touches user experience, frontend
architecture, product flows, landing pages, dashboards, onboarding, settings,
checkout, admin tools, AI copilots, agent workspaces, or generated interfaces.

## Default Direction

Aim for premium futuristic SaaS, not cyberpunk.

The product should feel:

- Modern.
- Clean.
- Intelligent.
- Trustworthy.
- Business-focused.
- Slightly futuristic.
- Easy to understand.
- Fast to use.

The interface should make the user feel like they hired a smart digital
employee. It should clearly show:

- What the AI is doing.
- What it found.
- What it recommends.
- What needs approval.
- What actions were completed.
- What business outcome was produced.

## Research Order

Start with real product UX, then add visual style.

1. Mobbin and Page Flows.
   Study real onboarding, dashboards, settings, pricing, checkout, search, and
   conversion flows before inventing a new pattern.
2. Godly, Land-book, and Awwwards.
   Use these for futuristic AI, SaaS, startup, and premium visual direction.
3. Spline, Rive, and Codrops.
   Use these for 3D, motion, and interaction ideas after the product flow is
   clear.
4. Apple Human Interface Guidelines, Material Design 3, Fluent 2, and Carbon.
   Use these to keep the interface usable, accessible, scalable, and familiar.

## Good Design Elements

- Dark/light adaptive interface.
- Clear dashboard hierarchy.
- Command palette.
- AI copilot panel.
- Subtle glass effects.
- Soft gradients.
- Strong typography.
- Animated workflows.
- Clear business actions.
- Excellent empty states.
- Clear onboarding.
- Human-readable status updates.
- Action-first interface.

## Avoid

- Heavy neon.
- Overdone cyberpunk style.
- Excessive 3D.
- Confusing navigation.
- Tiny text.
- Slow animations.
- Visual clutter.
- Too many gradients.
- Mystery-meat icons.
- Fancy UI that hides the real action.

## Agent Requirements

When implementing UI/UX work in a target product repo:

- Start from the actual user workflow before choosing visuals.
- Use real product UX references before visual inspiration references.
- Keep futuristic polish subordinate to clarity and speed.
- Make AI state explicit: thinking, working, waiting for approval, blocked,
  completed, failed, or needs review.
- Make risky actions require clear owner approval.
- Provide evidence behind recommendations.
- Verify desktop and mobile layouts.
- Confirm text does not overlap or truncate in primary flows.
- Confirm risk is not communicated by color alone.
- Confirm empty, loading, error, blocked, and success states exist where needed.
- Use animation to clarify state, not hide it.

## Final Principle

The best futuristic product design is not the flashiest one. It is a clean,
premium, AI-native business interface that feels futuristic because it is fast,
intelligent, animated, and action-oriented.
